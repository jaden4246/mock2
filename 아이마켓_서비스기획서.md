# 아이마켓 (KidsTrade) 서비스 기획서

> 아이들이 직접 사고 파는, 아이들만의 중고거래 플랫폼

---

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 아이마켓 (KidsTrade) |
| 서비스 유형 | 아동 특화 지역 기반 중고거래 웹 플랫폼 |
| 타깃 사용자 | 만 6세~13세 아동 (주 사용자), 보호자 (관리·승인 역할) |
| 핵심 가치 | 안전한 거래 환경 + 경제교육 + 지속가능한 소비 |

---

## 2. 문제 정의 (Problem Statement)

### 2.1 현황
- 국내 중고거래 플랫폼(당근마켓, 번개장터 등)은 **성인 중심**으로 설계되어 있어 아동이 직접 사용하기 어려움
- 아이들이 쓰다 남긴 장난감, 책, 학용품 등은 **버려지거나 방치**되는 경우가 많음
- 아동의 **금융·경제 교육** 기회가 일상 속에서 부족

### 2.2 Pain Points
- **아이(사용자)**: 안 쓰는 물건을 쉽게 처분하고, 원하는 물건을 저렴하게 구하고 싶다
- **부모(보호자)**: 자녀의 거래를 안전하게 감독하고 싶다. 낯선 성인과의 접촉이 걱정된다
- **사회적 관점**: 아동 소비재의 재활용률이 낮아 환경 부담이 크다

---

## 3. 솔루션

아이들이 **직접** 물건을 올리고 거래하되, **부모가 최종 승인**하는 구조의 안전한 중고거래 플랫폼

- 아이 전용 UI/UX (직관적, 시각적, 게임화 요소)
- 보호자 연동 계정 및 거래 승인 시스템
- 학교·동네 기반 지역 매칭
- 포인트·뱃지를 통한 경제 학습 동기부여

---

## 4. 핵심 기능 (Core Features)

### 4.1 아이 계정 (Child Account)
- 보호자 계정에 연동된 서브 계정으로 생성
- 닉네임, 프로필 캐릭터 선택 (실명·사진 노출 최소화)
- 학교명 또는 동네 인증 (보호자가 진행)

### 4.2 물건 올리기
- 카테고리: 장난감 / 책·학습지 / 문구·미술 / 스포츠·야외 / 옷·신발 / 기타
- 사진 업로드 + 간단한 설명 입력 (음성 입력 지원)
- AI 자동 카테고리 분류 및 적정가격 추천
- **보호자 게시 승인** 후 공개

### 4.3 거래 매칭
- 학교 단위 / 동네 반경(1~3km) 기반 매칭
- 채팅: 텍스트 + 스티커만 허용 (전화번호·개인정보 자동 차단)
- 거래 장소: 학교 앞, 커뮤니티센터 등 **안전 장소 목록** 제공 및 추천

### 4.4 보호자 대시보드
- 자녀의 등록 물품, 채팅 내용 열람
- 거래 승인 / 거부 알림
- 거래 완료 후 입출금 내역 확인 (용돈 연동 옵션)

### 4.5 경제 교육 요소 (Gamification)
- 거래 완료 시 **씨앗 포인트** 적립
- 판매·구매 횟수에 따른 **등급 뱃지** (새싹 → 열매 → 나무)
- 월간 절약 금액 시각화 ("이번 달 OOO원 아꼈어요!")
- 환경 기여 지수 ("탄소 OOkg 줄였어요!")

## 5. 사용자 흐름 (User Flow)

```
[회원가입]
보호자 계정 생성 → 자녀 계정 추가 → 동네/학교 인증

[물건 판매]
아이: 사진 촬영 → 설명 입력 → 가격 설정 → 등록 요청
보호자: 확인 → 승인 → 게시 완료

[물건 구매]
아이: 검색/탐색 → 찜하기 → 채팅 → 구매 요청
보호자: 승인 → 거래 장소 확인 → 거래 완료 확인

[정산]
현금 직거래 또는 연동 용돈 계좌로 처리
```

## 10. 개발 로드맵

### Phase 1 — MVP (0~6개월)
- [ ] 보호자·아이 계정 연동
